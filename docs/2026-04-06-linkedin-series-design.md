# LinkedIn 시리즈 설계 — DAN Research 잠재 고객 유입

## 기본 설정

- **타겟**: 증권사 LP/파생 데스크 의사결정자
- **목적**: 잠재 고객 유입 ("이 팀 괜찮네")
- **총 4편**, 주제별 독립 (단독으로도 읽힘)
- **링크**: 본문에 안 넣고, 첫 댓글에 `dan-research.co`
- **게시 빈도**: 미정 (글만 먼저 완성)

## 시리즈 구조

### 1편 — "트레이더의 시간은 어디에 쓰이고 있는가"

- **톤**: 짧고 임팩트 (400~600자)
- **역할**: 시리즈 입구. 공감 유발. 회사 소개 최소화.
- **흐름**: 종목 늘어남 → 관리 업무 폭증 → 매매/판단 시간 줄어듦 → "이 구조를 바꿔야 한다" → DAN Research는 이 문제를 시스템으로 풀고 있다 (한 줄)
- **핵심 메시지**: 트레이더가 운영이 아니라 매매에 집중할 수 있어야 한다

### 2편 — "시장조성, 종목이 300개면 사람이 다 볼 수 없다"

- **톤**: 케이스 스터디 (600~1000자)
- **역할**: 시장조성 모듈의 가치를 구체적 상황으로 전달
- **흐름**: 상황 묘사 (의무호가, 점수관리, 이론가 조정을 종목별로 수동) → 어디서 실수/병목 발생 → 자동화로 뭐가 바뀌는지 → DAN Research가 구축한 방식
- **핵심 메시지**: 반복 감시를 시스템이 하고, 사람은 예외만 본다

### 3편 — "ETF LP, 호가만 내면 끝이 아니다"

- **톤**: 케이스 스터디 (600~1000자)
- **역할**: ETF LP 모듈. NAV 차익 + MM + 통합헤지가 왜 하나여야 하는지
- **흐름**: 상황 묘사 (NAV 차익 포착, 스프레드, 헤지가 각각 따로) → 분리 운영의 비효율 → 통합 시스템으로 뭐가 달라지는지
- **핵심 메시지**: 호가·헤지·리스크를 분절하지 않고 하나의 시스템으로

### 4편 — "차익거래에서 사람이 매일 확인하는 것들"

- **톤**: 케이스 스터디 (600~1000자)
- **역할**: 차익거래 모듈. 공시·배당·스프레드 감시의 체계화
- **흐름**: 상황 묘사 (공시 확인, 배당 반영, 스프레드 변동 감시를 매일 수작업) → 놓치면 생기는 일 → 체계적 감시 시스템 → DAN Research가 구축한 방식
- **핵심 메시지**: 매일 반복되는 감시를 시스템으로 전환
